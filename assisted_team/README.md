## Deployment

Make sure Docker and Docker-compose are installed

Execute:

    docker-compose up -d

If everything works, then you'll be able to access main API endpoint at http://localhost:8080/api/ and Swagger UI at http://localhost:8081/

### Комментарии

Для бекенда был выбран Flask, поскольку в тестовой задаче, на мой взгляд, не предполагалось использование базы данных и не было нужны в полноценном развесистом фреймворке. Т.к. требовалось реализовать только несколько эндпоинтов, то легковесный Flask показался довольно подходящим решением.

В тестовых данных отсутствовали различные варианты маршрутов, поэтому, на текущий момент, приложение считает *только* по предложенному маршруту DXB-BKK.

Во многих местах в коде оставлены `TODO`-заметки, т.к. целиком охватить все возможные проблемы и подводные камни в рамках тестового задания невозможно, но хотелось бы обратить внимание, что проблемы подсвечены.

Поскольку основной юзкейс неизвестен, никаких специальных оптимизаций не делалось. В реальном проекте нужно будет проанализировать нагрузку и, кроме прочего, реализовать какое-либо кэширование (таймзоны аэропортов, единожды подсчитанные параметры перелетов, прочитанные XML файлы).

В качестве первой версии критерия оптимальности было добавлено просто некое произведение цены на длительность, но функционал вынесен в отдельынй метод и содержит несколько возможных идей по улучшению.

В процессе разработки нужды в Docker не было, но решил в конце добавить, чтобы можно было развернуть на любой машине "в один клик" и придать ощущение завершенности :)

## Тестовое задание в команду ассистеда (Python/Go)

В папке два XML – это ответы на поисковые запросы, сделанные к одному из наших партнёров.
В ответах лежат варианты перелётов (тег `Flights`) со всей необходимой информацией,
чтобы отобразить билет на Aviasales.

На основе этих данных, нужно сделать вебсервис,
в котором есть эндпоинты, отвечающие на следующие запросы:

* Какие варианты перелёта из DXB в BKK мы получили?
* Самый дорогой/дешёвый, быстрый/долгий и оптимальный варианты
* В чём отличия между результатами двух запросов (изменение маршрутов/условий)?

Язык реализации: `Go`
Формат ответа: `json`
По возможности использовать стандартную библиотеку.

Язык реализации: `python3`
Формат ответа: `json`
Используемые библиотеки и инструменты — всё на твой выбор.

Оценивать будем умение выполнять задачу имея неполные данные о ней,
умение самостоятельно принимать решения и качество кода.
